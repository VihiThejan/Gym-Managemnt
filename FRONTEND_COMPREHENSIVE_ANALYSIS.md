- ‚úÖ Multi-role authentication (Admin, Staff, Member)
- ‚úÖ Radio button role selection
- ‚úÖ Phone number input with country code (react-phone-input-2)
- ‚úÖ Real-time validation (mobile & password)

- ‚úÖ Registration links for all roles

**API Endpoints:**
```javascript

Staff:  POST /api/v1/staffmember/login
Member: POST /api/v1/member/login
```

**Authentication Storage:**
```javascript

```

**Navigation:**
- Admin ‚Üí `/Dashboard`
- Staff ‚Üí `/StaffDashboard`
- Member ‚Üí `/MemberDashboard`

**Validation Rules:**
- Mobile: Minimum 10 digits
- Password: Minimum 6 characters

---

### **2. Registration (Admin.jsx, staff.jsx, Member.jsx)**

**Common Validation Rules:**
- **Name:** First and last name, each capitalized
- **Mobile:** 11+ digits with country code
- **Email:** Valid email format
- **Password (Admin only):** 6+ chars, uppercase, lowercase, number, special char
- **Address:** 10+ chars, must contain letters and numbers

**Member-Specific Fields:**
- DOB, Gender, Height, Weight
- Package selection (Gold/Silver/Bronze)
- Username & Password

**Staff-Specific Fields:**
- DOB, Gender, Job Role
- Username & Password

**Admin Fields:**
- Name, Mobile, Password only

---

### **3. Password Recovery (Forgotpw.jsx, Resetpw.jsx)**

**Two-Step Process:**
1. **Forgot Password:**
   - Enter phone number
   - Request OTP via SMS
   - Verify OTP code
   
2. **Reset Password:**
   - Enter new password
   - Confirm password
   - Submit to backend

**API Endpoints:**
```javascript
POST /api/v1/auth/forgetpw  // Request OTP
POST /api/v1/auth/verify    // Verify OTP
POST /api/v1/auth/reset     // Reset password
```

## üìã **CRUD Operations Pattern**

### **Standard Component Structure**

Each module follows the same 3-component pattern:

#### **1. Create Component** (e.g., `Member.jsx`)
- Form with validation
- Submit ‚Üí `POST /api/v1/{module}/create`
- Reset/Cancel functionality
- Navigation back button

#### **2. Table Component** (e.g., `MemberTable.jsx`)
- Data fetching on mount ‚Üí `GET /api/v1/{module}/list`
- Ant Design Table with:
  - Search functionality
  - Edit button ‚Üí navigates to `/Member/:id`
  - Delete button ‚Üí `DELETE /api/v1/{module}/delete/:id`
  - Confirmation modal for delete
- Real-time data refresh after operations

#### **3. Edit Component** (e.g., `EditMember.jsx`)
- Fetch data by ID ‚Üí `GET /api/v1/{module}/:id`
- Pre-populate form fields
- Update ‚Üí `PUT /api/v1/{module}/update/:id`
- Same validation as Create

const ProtectedRoute = ({ children, allowedRoles }) => {
